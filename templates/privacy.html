Privacy Policy

Jarvis Student Assistant · a personal/family-use application

This application is a personal student-productivity assistant built for private, family use — not a public product. This page exists so that connected third-party services (such as WHOOP) have somewhere to point users regarding data handling.

What data this app stores

  • Account credentials — a login password for the app itself, stored as a salted hash.
  • Calendar data — assignment, personal, sports, and job-schedule calendar feeds you provide (iCal URLs), fetched to build schedules and briefings.
  • Canvas data — if you provide a Canvas API token, course names, grades, and assignment details are fetched to give the assistant fuller context.
  • WHOOP data — if you connect a WHOOP account, recovery score, sleep, strain, HRV, and resting heart rate are fetched via WHOOP's API and cached briefly to display on the dashboard and inform the assistant's advice.
  • Tasks, projects, and notes — anything you type into the app's task, project, or notes features.
  • Chat history — conversations with the assistant, stored so it can recall prior context.

How this data is used

All of the above is used solely to power this application's own features — daily briefings, evening debriefs, chat, dashboards, and scheduling. Relevant snapshots of this data (e.g. today's assignments, recent WHOOP recovery) are sent to the Anthropic Claude API to generate responses. Data is not sold, shared with advertisers, or used for any purpose beyond operating this application.

Third-party services

Depending on which integrations you configure, data flows to and from:

  • Anthropic (Claude API) — to generate chat responses and briefings
  • WHOOP — to fetch recovery/sleep/strain data via OAuth2, if connected
  • Canvas / your school's LMS — to fetch assignments and grades, if configured
  • Calendar providers (Google, etc.) — to read calendar feeds you provide

Each integration is opt-in and can be disconnected at any time from the app's Settings page.

Data storage & retention

Data is stored in a private PostgreSQL database associated with this application's hosting environment. OAuth tokens (e.g. for WHOOP) are stored server-side and never exposed to the browser. You can disconnect an integration at any time, which removes its stored tokens; deleting other stored data can be requested from the site administrator.

Contact

This application has a single administrator/owner. Questions about data handling can be directed to whoever provided you access to this app.
